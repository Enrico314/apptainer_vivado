# Appatiner for vivado 
This repo contains an apptainer recpie for Vivado. This repo is mainly focused on how to build the container.

## How to use 
### Default
Make sure to have a resource`folder containing the .tar file from xilinx-amd webside. The file must be downlowaded separatly as it is behind a login wall.

**Warning**: by default all the EULAs are accepted in the build script.

The folder structure should look like this to have the default build succesfull
```bash
├── build.sh
├── resources
│   └─── FPGAs_AdaptiveSoCs_Unified_SDI_2025.2_1114_2157.tar # as an example. Any tarball containing vivado should work.
└─── vivado.def
```

To build naivly:
```bash 
./build.sh 
```

Specify your configuration parameters in the `vivado.def` file if necessary. You can customize the type of installation by changing `xetup` file autogenerated in the `vivado.def` recipe.

### User specific options
The recipe can be used with `build-arg` options. The following are available with their default values:
```bash
INSTALL_TARGET_DIR="/opt/xilinx"
HOST_TOOL_ARCHIVE_NAME="FPGAs_AdaptiveSoCs_Unified_SDI_2025.2_1114_2157.tar"
XILINX_INSTALL_TMP="/resources/xilinx/xilinx_installer/"
XSETUP_FILE="install_config.txt"
```
The user can customize them as needed. 


#### Futher notes:
 - The container is quite nice: ~20G. The building process requires quite a bit of space ~ 200G.
 - Building the container is time consuming < 20 min (depending on the hardware).
 - The build script has been tested for the Vivado2025.2 version. Other editions might require adjustments.
 - This version is not tested with driver support. Some additional tweks might be required to use this container to directly interact with specific driver.


#### How to run the container 
To run the container with binding can use:
```bash
apptainer run \
	--bind /host/src:/src \
	--bind /host/work:/work \
	vivado.sif
```

Refer to the help of the container:
```bash 
apptainer run-help <image.sif>
```
